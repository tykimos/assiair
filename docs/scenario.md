## 시나리오 5: 공문 이메일 발송 요청

- **사용자**: "행사 참석을 위한 공문을 이메일로 보내주세요"
- **분류**: document (official_letter)
- **스킬**: document-request
- **워크플로우**: document-request
- **수집 정보**: 이름, 소속(회사/기관명), 직위, 이메일
- **처리 흐름**:
  1. 필요 정보 확인 (Runtime Context의 UserState에서 이름/이메일 있으면 건너뜀)
  2. 부족한 정보 요청
  3. 공문 내용 생성 (행사명, 일시, 장소 포함)
  4. `generate_document`로 PDF 생성
  5. 사용자 확인
  6. `send_email`로 발송
- **후속 버튼**: "다른 문서 요청"

## 시나리오 8: 복합 요청 (Q&A → 문서)

- **사용자**: "행사 안내 좀 해주시고, 공문도 보내주세요"
- **분류**: event → document
- **스킬**: event-qa → document-request
- **워크플로우**: orchestrator가 event-qa → document-request 순으로 스킬 체이닝
- **처리 흐름**:
  1. 먼저 행사 안내 응답 (event-qa 스킬)
  2. 이어서 공문 발송 절차 진행 (document-request 스킬)
  3. 필요 정보 수집 후 문서 생성 및 발송
- **후속 버튼**: "다른 문서 요청"
